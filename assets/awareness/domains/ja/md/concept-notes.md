# Concept Notes に関する調査報告

## 1. この report の問い

本 report の問いは、Concept Notes 群が `awareness-space` において何を支えるのか、そしてなぜ guide や主要 report とは別の層として必要なのかである。

## 2. Concept Notes の位置づけ

Concept Notes は、意識モデルを補助する概念や境界事例を保持するノート群である。

ここに含まれるのは、たとえば次のような論点である。

- 内在化された関係
- 生存軸の衣が剥がれる体験
- 境界ケース
- 集団 Withhold
- 信頼の定義仮説
- 信頼を記述するための軸
- 測定設計原則

これらは、主要 report の本文にそのまま入れるには早すぎるが、捨てるには重要すぎる論点である。

## 3. このノート群で見えること

第一に、Concept Notes は主軸の議論の盲点を補う。4層モデルや Withhold だけでは扱いにくい境界事例、関係構造、測定の問題を追うことができる。

第二に、Concept Notes は「まだ未確定だが重要な論点」を保持する。信頼や間主観性のように、単一定義へ閉じにくい領域では特に有効である。

第三に、Concept Notes は将来の report 候補の母体でもある。現時点では入口 report に留めるが、読み込みが進めば個別の report に昇格しうる。

## 4. overview との違い

- overview は「どこから読めばよいか」を案内する
- Concept Notes report は「この束が何のためにあるか」を説明する

index だけでは一覧に留まるが、report にすることで読者はこの束の役割を理解しやすくなる。

## 5. まだ決めていないこと

- どの Concept Note を先に個別 report 化するか
- 一覧中心で見せるか、主題別に束ね直すか
- 信頼系ノートを 生存-信頼軸 report の補助線としてどこまで統合するか

## 6. まとめ

Concept Notes は、`awareness-space` における探索的な補助層である。主軸の report を補強しつつ、まだ未確定な論点を保持し、次の report 候補を育てる役割を持つ。
