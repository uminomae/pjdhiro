# M1 consciousness OS に関する調査報告

## 1. この report の問い

本 report の問いは、`awareness-space` において M1 を何として残すべきかである。結論から言うと、M1 は独立した完成理論というより、4層モデル、生存-信頼軸、Withhold をつなぐ基盤文章群として読むのが妥当である。

## 2. 現在の位置づけ

M1 は旧称としては「意識OS」を指していたが、`awareness-space` ではその名称をそのまま中心概念として採用しない。現在は、4層モデルの原著文書群を束ねる source 束として扱う。

ここで重要なのは、M1 を捨てることではなく、役割を言い換えることである。4層モデル report は現在の reader-facing な最小フレームを説明する。一方で M1 report は、そのフレームがどの文章群の上に立っているかを示す。

## 3. M1 が支えているもの

第一に、M1 は意識を静的な箱ではなく、作動の連鎖として読む基盤を与える。Part 1 から Part 5 までの流れを通すと、身体状態、ズレの選別、生存-信頼評価、保持という順序がどのように組み立てられてきたかが見える。

第二に、M1 は 4層モデル、生存-信頼軸、Withhold を別々の断片としてではなく、同じ系の中で接続する。4層モデル report だけを読むと骨格は掴めるが、M1 の文章群を読むと、その骨格がどの論理展開から出てきたのかを追える。

第三に、M1 は `awareness-space` における source reading の保存層として機能する。読者向け report へ圧縮する前の文章的な厚みを保持しておくことで、後から Layer 2 や Layer 3 を再精密化するときの足場になる。

## 4. 現時点での読み方

現在の M1 は、次の 5 パートを持つ source 束として読むのがよい。

1. Part 1: 導入
2. Part 2: 内受容感覚
3. Part 3: 予測-誤差ループ
4. Part 4: 生存-信頼評価
5. Part 5: Withhold

この並びにより、M1 は「旧ラベル」以上の意味を持つ。名称そのものを理論の中心に据える必要はないが、この束を保持することで、4層モデル report がどの source から再構成されたかを示せる。

## 5. 4層モデル report との違い

- 4層モデル report は、現在の最小フレームを説明する reader-facing report である
- M1 report は、そのフレームを支える基盤文章群の役割を説明する source report である

したがって両者は重複ではない。4層モデルが「何を言っているか」を担い、M1 は「その骨格がどこから来たか」を担う。

## 6. まだ決めていないこと

- `M1` という名称を公開導線でどこまで前面に出すか
- Part 群を今後も 1 束として読むか、層ごとにさらに分解するか
- Withhold を M1 の終点として残すか、条件付き機能として再配置するか
- M1 と神経現象学 / 発達心理学の読解をどの粒度で接続するか

## 7. まとめ

`awareness-space` における M1 は、独立理論名として固定するより、4層モデルの原著文書群を束ねる source 束として扱うのが適切である。役割は、現在の report 群の背後にある文章的土台を残し、4層モデル、生存-信頼軸、Withhold の接続経路を可視化することにある。
