# Awareness Model in Dialogue with Neurophenomenology and Psychology

## Purpose

This guide presents the awareness model as an exploratory framework to be read in relation to neurophenomenology and psychology.
It does not claim to offer a completed theory.
Its purpose is to clarify the central hypothesis, the comparison axes used in the research, the current state of the argument, and the questions that remain unresolved.

## Executive Summary

- The current starting hypothesis is that experience can be interpreted along two orientations: survival and intersubjectivity.
- The four-layer model is used to describe awareness as a process from bodily state to possible revision of response.
- The main challenge is not a lack of material but the unfinished integration of neurophenomenological and psychological lines of inquiry.

## Model Overview

![Survival-Trust Axis overview](https://uminomae.github.io/pjdhiro/assets/awareness/img/svg/topics/en/survival-trust-axis-01-overview-svg.svg)

![Four-Layer Model overview](https://uminomae.github.io/pjdhiro/assets/awareness/img/svg/topics/en/four-layers-01-overview-svg.svg)

The current working hypothesis is that the direction of experience can be read through two axes.

- Survival axis: survival, safety, defense, self-protection
- Trust axis: relationship, trust, belonging, intersubjectivity

These axes are not a classification of emotions.
They are a hypothetical coordinate system for asking how discrepancy is interpreted and where it is directed.

At the same time, the model treats awareness as a four-layer process.

1. Layer 0: bodily state
2. Layer 1: discrepancy between prediction and reality
3. Layer 2: directional interpretation of discrepancy
4. Layer 3: room for response to remain revisable

## Research Basis and Reference Fields

From neurophenomenology, the project takes the question of how first-person experience can be described in workable units.
That currently includes embodiment, foregrounding of discrepancy, first-person descriptive units, and prereflective awareness.

From psychology, the project takes the question of how far the survival-trust hypothesis can be tested, especially on the relational side.
Developmental psychology, attachment theory, internal working models, trust research, intersubjectivity, and repairability are therefore treated as major reference areas.

Earlier source bundles are treated as historical material rather than as present-day canonical texts.
The current public frame is organized around the survival-trust axis and the four-layer model.

## Current View

What currently appears most plausible is to read awareness not as static content but as a sequence from bodily state to salience, interpretation, and response margin.

This allows emotion, cognition, and action-preparation to be described as parts of one chain rather than as isolated boxes.
Once the survival-trust axis is added, the same discrepancy can be distinguished according to whether it is being read mainly as a safety issue or mainly as a relationship issue.

The present material also suggests that the survival-trust axis and the four-layer model are not unrelated fragments.
They appear to support the same problem space at different levels of granularity.

## Key Limitations and Next Questions

There are already plausible structural hypotheses and a supporting body of material.
However, the point at which neurophenomenology and psychology should be integrated is still unsettled.
So is the question of how strongly any given claim should be stated in public-facing documents.

The main open questions are:

- How rigorously can the boundary between survival-oriented and trust-oriented problems be defined?
- Is the four-layer model sufficient as a minimal descriptive framework for awareness?
- How neutral can Layer 1 discrepancy be described?
- How should neurophenomenological clusters be returned to the four-layer model?
- How should developmental-psychological clusters be returned to the trust axis?
- To what extent can neurophenomenology and developmental psychology be integrated into one structure?
- Where should observability be located?
