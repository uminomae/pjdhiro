# 意識モデルを観察の道具として使う

## この guide が何であり何でないか

人の反応を前にすると、
私たちはすぐに「落ち着いていない」「説明が足りない」「感情的すぎる」と評価しがちです。
しかし現場で必要なのは、
良し悪しを急いで決めることより、
何が起きているかを観察することです。

この guide は、そのためのモデル解説です。
実践マニュアルでも診断ラベルでもなく、
現場で起きる反応を構造として見るための足場を渡す文書として書いています。
同時に、ここで扱うモデルは完成理論ではなく、
調査を踏まえてなお探索中の仮説であることを前提にします。
また、この guide 自体は summary であり、
詳細な論拠は REPORTS、調査概要、設計メモに分かれています。

## モデルの中心仮説

このモデルは、意識を4層の流れとして見ます。

- Layer 0: 体の状態
- Layer 1: 予測と現実の差
- Layer 2: その差が何に関わる問題として読まれるか
- Layer 3: 反応が固定されず、見直されうる余地

特に Layer 2 を読む仮説として、
生存-信頼軸を置いています。

- 生存軸: 安全、防衛、立場、失敗回避
- 信頼軸: 信頼、関係、理解されること、所属

たとえば沈黙ひとつを見ても、
失敗を避けようとしているのか、
関係を壊したくないのかで、
観察の仕方も関わり方も変わります。

## その仮説を支える調査

この見方は現場感覚だけで作っているわけではありません。
調査の中心に置いているのは、
神経現象学と心理学です。

神経現象学には、
一人称経験をどう記述できるかを学ぶために接続しています。
心理学には、
愛着、発達、信頼、間主観性のような論点を通じて、
関係側の問題をどこまで構造的に扱えるかを確かめるために接続しています。

さらに、既存の文章資産は歴史的 source 束として読み直し、
現在の観察枠を従属させない前提で補助線として使っています。

ただし、ここでも段階差があります。
生存-信頼軸、4層モデル、Concept Notes には
読者向け report があります。
一方で、神経現象学や発達心理学は
重要な中心調査軸ですが、
現時点では reader-facing な専用 report より、
設計メモの段階が先行しています。

`抱持` は、現場記述に使える仮ラベルではあるものの、
独立概念としては粗すぎると判定され、現在は概念監査中です。

## 現場で何が見えるか

### 教育

学習が止まっているように見えるとき、
内容理解だけを疑うと見誤ることがあります。
緊張が強いのか、
予想との差が大きいのか、
関係の不安が前に出ているのかを分けて見る必要があります。

### 支援

支援場面では、
反応の大きさだけを見ると「不安が強い」で終わりがちです。
しかし、その不安が安全の問題なのか、
関係の問題なのかで、
支援者の立ち位置は変わります。

### チーム

チームでは、個人の反応と場の反応が重なります。
ある人の防衛的な反応が、
別の人には関係不安として返り、
場全体の緊張を高めることがあります。
意識モデルは、その連鎖を観察する補助線になります。

## どこまで進んでいるか

現時点では、
体の状態、差の前景化、方向づけ、反応の余地という流れで
反応を読む見方はかなり有力に見えています。
ただし、source は厚くても、
それを統合理論としてどこまで整理できるかはまだ途中です。

公開している構成要素レポートも、
完成版ではなく公開ドラフトの段階です。
つまり、現場で使える観察の補助線としては一定の論拠がありますが、
神経現象学や発達心理学との接続まで reader-facing に出揃った状態ではありません。
現場への接続可能性は見えていますが、
どこまで一般化して使えるかはまだ検証しきれていません。

## 何がまだ未解決か

- 生存軸と 信頼軸の境界をどこまで明確にできるか
- 神経現象学と心理学の役割分担をどう設計するか
- 反応の見直しの余地をどこまで構造として扱えるか
- 観察の補助線を、どこまで検証可能な形に下ろせるか

この見方は、現場で使える可能性を持っています。
しかし、だからといって確定した方法論になったわけではありません。
